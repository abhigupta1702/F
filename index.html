    A component framework for JavaScript applications

    ƒ helps you build large, modular apps.

    Dead simple inheritance, nested components, view rendering, and easy REST integration.

    ƒ doesn't reinvent the wheel.

    Using Backbone, jQuery, and Handlebars, ƒ provides a simple way to modularize large JavaScript applications.

    ƒ is classy.

    ƒ provides an inheritance model, Class. Based on Douglas Crockford's new_constructor, Class lets you create building blocks that you can extend and integrate.

    Getting started is easy.

    1. Create a namespace:

    Nobody likes global variables. Keep it neat with a namespace.

    var app = {};

    2. Build your components

    Anything that needs a model, view, or controller can be a component.

    app.Welcome = new Class({ toString: 'Welcome', extend: F.Component, construct: function(options) { _.extend(options, { // Render this component with our template template: this.ItemTemplate, // Let events call this component's methods component: this }); // Create your view, no need to render! this.view = new F.View(options); }, // Store templates and events here so they can be overridden Template: Handlebars.compile( '<strong>Welcome!</strong>'+ '<button class="sayHi">Say hi!</button>' ), events: { 'click .sayHi': 'sayHi' }, // Define your methods sayHi: function() { alert(this.toString()+' says hi!'); } });

    3. Setup your app

    Your app is just another component.

    app.Main = new Class({ toString: 'Main', extend: F.Component, options: { visible: true }, construct: function(options) { // Config is always passed to the constructor console.log('%s: started. Options:', this, options); _.extend(options, { // Add this component to body of the page parent: document.body, // Render this component with our template template: this.Template, // Let events call this component's methods component: this }); // Create a view for the app // Render right away if you use selectors below this.view = new F.View(options).render(); // Add your components this.addComponent(new app.Welcome({ // show the component immediately visible: true, // add the sub-component to the div.welcome element parent: this.view.$('.welcome') })); }, destruct: function() { // Superclass destructors are called for you, and F // will call destruct() on all your components for you console.log('%s: shutting down', this.toString()); }, // tore the template in the prototype so it can be overridden Template: Handlebars.compile( '<h1>Killer App</h1>'+ '<div class="welcome"></div>' ) });

    3. Create an instance of your app

    Fire it up!

    var myApp = new app.Main();

    Models, views, and what?

    Let's avoid the buzzword game and explain we we're talking about.

    Models

    Models are a local representation of an object or database entry.

    tl;dr: ƒ uses vanilla Backbone models.

    Templates

    Templates are HTML structure and template code. Templates control the presentation of your models on the page.

    tl;dr: ƒ uses vanilla Handlebars templates.

    Views

    Views render a model with a given template, putting your data into your HTML structure. Views can be shown or hidden.

    tl;dr: ƒ uses vanilla Backbone views.

    Components

    Components are logical units of application code. They have methods, model(s), view(s), and sub-components. They're nested; a component can consist of many components that it connects together via events and method calls.

    tl;dr: Components are mini-apps that orchestrate interaction between sub-components.
